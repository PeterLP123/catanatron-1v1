# Documentation

The documentation is deliberately small and specific to this fork.

- [Rules reference](RULES.md): the exact 1v1 settings implemented by the simulator.
- [Training and evaluation](TRAINING.md): hashed data, BC losses, distillation, PPO, evidence suites, artifacts, and troubleshooting.
- [GPU experiment backlog](GPU_EXPERIMENT_BACKLOG.md): generated queue, executable evidence gates, inputs, resources, and stop rules.
- [Project plan](PLAN.md): current evidence, direction, promotion gates, and fallback research track.
- [Results log](RESULTS_LOG.md): provisional historical context and the corrected evidence ledger.
- [Tracked model evidence](results/README.md): compact validated promotion/final summaries.
- [Architecture](ARCHITECTURE.md): runtime, model-schema, evidence, dependency, and extension boundaries.
- [Project README](../README.md): installation and the shortest working path.
- [Upstream notice](../NOTICE.md): provenance, license, and third-party status.

For generic engine behavior not changed by this fork, consult the [upstream Catanatron source and documentation](https://docs.catanatron.com/). Upstream web, deployment, and documentation assets are not mirrored here.
